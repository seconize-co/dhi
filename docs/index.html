Seconize Dhi

Runtime Security for AI Agents

Dhi protects AI agents in real time with kernel + application visibility. It helps teams detect and stop secrets leaks, data exfiltration, prompt injection, and budget abuse.

Install and Try Technical README

Problem Statement

AI agents can call tools, access credentials, and transmit sensitive data over network APIs. Traditional app-layer controls miss runtime behavior, while incident response often comes too late.

  • Credential leaks in prompts, logs, or outbound requests
  • PII exfiltration through model/tool interactions
  • Prompt injection and jailbreak attempts
  • Unbounded token spend and runaway loops

Core Capabilities

Secrets + PII Detection

Pattern and policy-driven detection for sensitive data exposure with configurable actions.

Prompt Attack Defense

Detection of prompt injection and jailbreak signatures across agent traffic and interactions.

Budget and Abuse Controls

Controls for spend ceilings, suspicious usage spikes, and noisy automation loops.

Linux eBPF SSL Visibility

HTTPS runtime inspection via eBPF SSL hooks on Linux, without MITM CA certificates.

Policy Enforcement

Run in log, alert, or block mode with operational safeguards.

Ops Integrations

Prometheus metrics, webhook/Slack alerting, and operational reports for security teams.

Use Cases

Secure AI Coding Assistants

Monitor and control outbound interactions from coding agents in developer environments.

Production Agent Guardrails

Apply organization policy to autonomous workflows handling customer or regulated data.

Compliance Evidence

Generate alerts and reports that support incident investigations and audit requirements.

Next Steps: Install and Try

  1. Install from GitHub Releases (Linux): 
    curl -fsSL https://raw.githubusercontent.com/seconize-co/dhi/main/scripts/install-linux-release.sh | bash -s -- v0.1.3
  2. Start Dhi: 
    sudo systemctl start dhi
sudo systemctl status dhi
  3. Validate runtime endpoints: 
    curl -s http://127.0.0.1:9090/health
curl -s http://127.0.0.1:9090/api/stats | jq .
